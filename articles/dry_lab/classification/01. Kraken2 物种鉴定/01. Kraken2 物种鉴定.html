01. Kraken 2 自建库及鉴定

目的与输入输出

当我们拿到一个新的菌株、环境样本或未知序列时，常常需要快速判断它更接近哪些已知分类单元。Kraken 2 通过 K-mer 与分类数据库进行快速匹配，适合对测序 reads 或组装序列进行物种组成鉴定。

如果已有合适的 Kraken2 数据库，可以直接鉴定；如果官方数据库太大、覆盖不理想，或者只想针对一批参考基因组做精细化判断，也可以用自己的 .fasta/.fna 序列自建数据库。

本文配套脚本 Kraken_2.py 将建库和鉴定整合成一个交互式流程：输入参考 FASTA、taxdump.tar.gz、TaxID 解析方式和待鉴定序列后，自动完成建库、运行 Kraken2 并输出报告。

需要准备的数据

  1. taxdump.tar.gz：NCBI taxonomy 数据，至少需要其中的 names.dmp 和 nodes.dmp。
  2. 参考序列：用于自建库的 .fa、.fasta、.fna 或对应 .gz 文件。
  3. TaxID 信息：可以手动输入，也可以使用 nucl_gb.accession2taxid、nucl_wgs.accession2taxid 等映射文件，或者直接使用已经包含 kraken:taxid|TaxID 的 FASTA header。
  4. 待鉴定序列：单端或双端 .fastq/.fq，也支持 .fasta/.fna。

环境检查

运行脚本前，请确认当前环境中可以调用 kraken2 和 kraken2-build：


# 查找二进制文件路径
which kraken2
which kraken2-build

# 检查软件版本
kraken2 --version

# 如果使用 Conda 管理环境
conda list | grep kraken2

数据库与 TaxID 解析

Kraken2 数据库不是一个单独文件，而是一个目录。构建完成后，目录中至少应包含 hash.k2d、opts.k2d、taxo.k2d 三个核心文件。脚本会在建库完成后检查这些文件是否存在。

自建库的关键是让 Kraken2 知道每条参考序列对应哪个 NCBI TaxID。脚本提供三种方式：

方式一：手动输入 TaxID

适合小型自建库，例如只有几个参考基因组，且你明确知道每个物种的 NCBI TaxID。脚本会验证 TaxID 是否存在于 taxdump.tar.gz 解压出的 nodes.dmp 中，并生成带标记的 FASTA：


>contig_1 kraken:taxid|12345
ATGC...

生成后的文件会放在数据库目录下的 library_with_taxid 文件夹中，再由 kraken2-build --add-to-library 加入数据库。

方式二：使用 accession2taxid 映射文件

适合使用 NCBI 原始参考序列，并且 FASTA header 保留了 accession 编号的情况。可以输入 nucl_gb.accession2taxid、nucl_wgs.accession2taxid 或对应的 .gz 文件。脚本会自动复制或解压到数据库的 taxonomy 目录。

这种方式要求参考 FASTA 的 header 中 accession 信息足够完整，否则 Kraken2 可能无法把序列对应到 TaxID。

方式三：FASTA header 已包含 TaxID

如果你的参考序列已经写好了 kraken:taxid|TaxID，可以直接选择该模式。脚本会先检查每个 FASTA 是否含有 kraken:taxid| 标记，然后直接加入 library。

交互式脚本流程

下载脚本后运行：


python Kraken_2.py

脚本会依次询问以下内容：

  1. 是否使用已有 Kraken2 数据库。如果选择是，只需输入数据库目录。
  2. 如果选择自建库，输入数据库输出目录。
  3. 输入参考 FASTA/FNA 文件、目录或 glob；多个路径可以用逗号分隔。
  4. 输入 taxdump.tar.gz 路径。
  5. 选择 TaxID 解析方式：手动 TaxID、accession2taxid 映射文件、或已有 kraken:taxid| header。
  6. 输入建库线程数，脚本自动执行 kraken2-build --add-to-library 和 kraken2-build --build。
  7. 输入待鉴定序列路径，脚本自动识别单端或双端样本。
  8. 输入鉴定线程数，生成 Kraken2 输出文件。

脚本支持输入单个文件、目录、通配符路径，也支持用逗号分隔多个路径。目录会递归搜索常见 FASTA、FASTQ 后缀。

结果文件

脚本会在当前页面同目录下创建类似 03_Kraken2_鉴定结果_YYYYMMDD_HHMMSS 的结果目录。每个样本主要输出：

  1. 样本名_report.txt：Kraken2 分类报告，用于查看各分类层级的 reads 占比。
  2. 样本名_kraken.out：每条 reads 或序列的分类结果。
  3. 样本名.log：Kraken2 运行时的标准错误输出，排查报错时优先查看。

自建库过程中，数据库目录下还会生成 build.log，记录 kraken2-build 的运行信息。

Kraken 输出及可视化

Kraken2 的 report.txt 是以 Tab 分割的 6 列数据，每一列代表的含义如下：

  1. 第一列 (Percentage)：分配给该物种（或该分类层级）的 Reads 占总 Reads 的百分比。
  2. 第二列 (Reads Covered)：分配给该层级及该层级以下所有子分类的总 Reads 数量。
  3. 第三列 (Direct Reads)：直接分配给该特定层级，而无法继续细分到更低层级的 Reads 数量。
  4. 第四列 (Rank Code)：分类级别代码。常见的有：U (未分类), R (根), D (域), K (界), P (门), C (纲), O (目), F (科), G (属), S (种)。
  5. 第五列 (TaxID)：NCBI 官方的物种分类编号。
  6. 第六列 (Name)：带有缩进的物种科学名称（缩进代表层级深度）。
