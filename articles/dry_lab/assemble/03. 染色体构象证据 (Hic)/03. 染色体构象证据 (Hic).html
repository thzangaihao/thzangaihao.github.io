03. 染色体构象证据（Hi-C）

前言

高通量染色体构象捕获（High-throughput Chromosome Conformation Capture，Hi-C）通过交联、酶切、邻近连接和高通量测序，记录细胞核内不同 DNA 片段的空间接触。位于同一条染色体且线性距离较近的区域通常具有更强的接触信号，因此这些数据不仅能研究染色质三维结构，也能为基因组组装提供远距离连接证据。

常规 Hi-C 使用双端测序：同一 read pair 中的 R1 和 R2 分别比对到组装序列，两端的基因组坐标共同构成一条 Hi-C 联系。将一对 reads 的两端独立比对，跨过连接位点的嵌合 read 则可能产生软剪切或分段比对。

本文使用 BWA-MEM、samtools、Picard 和 YaHS，将长读长组装得到的 contig 自动聚类、排序、定向并连接为 scaffold。该过程称为 scaffolding（挂载），并不校正 contig 内部的碱基错误。

各步骤的作用

  1. FastQC/MultiQC：汇总 read 长度、碱基质量、GC、接头及重复水平，只做质控，不修改原始 FASTQ。
  2. BWA-MEM：使用 -5SP 将每组 Hi-C paired-end reads 比对到原始组装，并写入 lane 级 read group。
  3. samtools：过滤 unmapped、secondary 和 supplementary alignment，按坐标排序后合并所有 lane。
  4. Picard：统一标记 PCR/光学重复。脚本设置 REMOVE_DUPLICATES=false，即保留记录但添加 duplicate flag。
  5. YaHS：依据 contig 间 Hi-C 联系完成错误连接检查、聚类、排序、定向和 scaffold 构建。

文件用途是否为核心结果

最终 FASTA 通常通过若干个 N 连接原始 contig；AGP 则是追踪这些连接关系的权威记录。原始 FASTQ 和原始 FASTA 均不会被修改。

脚本及环境

交互式脚本会递归搜索 .fq/.fastq 及其 gzip 文件，识别 R1/R2、READ1/READ2 或 _1/_2 命名，列出配对后供用户选择。脚本支持断点恢复：已经完整生成的 lane BAM、合并 BAM 和最终 YaHS 结果会被跳过；Picard 只有成功结束后才写入完成标记，避免将 OOM 后的残缺 BAM 当成有效结果。

断点恢复

流程意外中断后，不需要专门指定“恢复模式”。再次运行同一脚本，选择相同的 FASTQ 配对和参考 FASTA，并将 --outdir 指向上一次的输出目录，脚本就会检查已有结果并从第一个未完成步骤继续。

conda activate hic-scaffold

python hic_scaffold.py \
    --search-dir /path/to/HIC \
    --fasta /path/to/assembly.fa \
    --outdir /home/thz/Analysis/2026_06_22_rice/04_hic \
    --threads 64 \
    --java-memory 64g \
    --motif GATC

恢复时必须保持输入数据、FASTA 和输出目录一致。脚本会按照以下规则判断是否跳过：

  • 参考序列的 .fai 和 BWA 索引存在时，不再建立索引。
  • qc/.multiqc_complete 存在时，不再运行 FastQC/MultiQC。
  • 某个 bam/*HIC_L*.sorted.bam 存在且非空时，跳过对应 lane 的比对。
  • bam/hic.merged.sorted.bam 存在且非空时，跳过 BAM 合并。
  • 只有 bam/.hic_dedup_complete 存在时，才认为 Picard 已完整成功；若上次因 OOM 中断且没有该标记，脚本会清理残缺的 dedup 输出并重新运行 Picard。
  • yahs/hic_scaffolds_final.fa 存在且非空时，认为 YaHS 已完成，否则重新运行 YaHS。

结果检查与解释

YaHS 输出的是自动推断的 scaffold，并不意味着每条 scaffold 已经可以直接命名为某条染色体。应结合预期染色体数、scaffold 长度、Hi-C 接触图、共线性或遗传图谱检查是否存在错误连接、倒置和错误拆分。还应在挂载前后比较总长度、N50/L50、BUSCO 完整度和序列条数；scaffolding 通常提高连续性，但不应凭空增加大量非 N 碱基。

Hi-C 互作分析

自动挂载与互作矩阵分析应拆成两个可独立复现的流程。挂载以原始 contig FASTA 为参考，目标是推断 contig 之间的组织关系；互作分析则以最终 hic_scaffolds_final.fa 为坐标，目标是生成 scaffold 层级接触矩阵并检查主对角线是否连续。

正式留档时可以将 Hi-C FASTQ 重新比对到最终 FASTA，再使用 pairtools、cooler 或 Juicer Tools 生成 .mcool/.hic。若只需快速检查，也可以复用 YaHS 产生的 .bin、最终 .agp 和原始 FASTA 的 .fai，通过 YaHS 附带的 juicer pre 完成坐标转换，无需重新比对。

方案 A（正式、坐标直观）
Hi-C FASTQ + hic_scaffolds_final.fa
        └── BWA → pairtools → cooler → final.mcool → HiGlass

方案 B（快速复用 YaHS 中间结果）
hic.bin + hic_scaffolds_final.agp + assembly.fa.fai
        └── juicer pre → Juicer Tools → final.hic → Juicebox

理想热图在每条主要 scaffold 内具有连续且强的主对角线，信号随线性距离增加而衰减，不同染色体之间的背景较弱。明显的断点、蝴蝶形、交叉或远离对角线的异常块状信号通常值得进一步人工检查。
