01. illumina 二代组装项目

1. 前言

这篇博客适用于由 illumina 二代测序平台产生的短读段数据在无参考证据文件时的从头组装（de novo组装）。因为组装流程已经被脚本高度自动化，这篇文章简单讲一下具体流程，环境搭建等，并解读软件的输出报告和意义。简单言之，我们是将公司给我们的.fastq文件组装出.fasta文件。

2. 环境搭建与软件安装

创建并激活专属 Conda 环境：


# 1. 创建基于 Python 3.10 的新环境
conda create -n assembly_env python=3.10 -y

# 2. 激活刚刚创建的环境
conda activate assembly_env

安装核心软件，这三个软件都托管在生物信息学的专属频道 bioconda 上。为了解决依赖问题，我们同时指定conda-forge频道。


conda install -c bioconda -c conda-forge fastp spades quast -y

fastp 用于数据的质控。你应该在收到公司递交的文件后看一眼质控报告，但无论如何，我们自己再洗一遍数据是最稳妥的做法，洗数据已经包含到流程里面了。fastp 进行接头修剪、低质量序列剔除，并生成 HTML 质控报告，查收并确定质量过关。

spades 专门针对单细胞和中小型基因组（如细菌、真菌）优化的 De novo 组装器。脚本中开启--fungal模式会专门针对真菌的重复序列进行优化。

quast 用于组装后的质控，会报道各种指标，例如 N50，我们会面后讲如何查看这些指标。

使用以下命令查看命令查看版本并验证安装：


fastp --version
spades.py --version
quast.py --version

4. 结果解读

我们输出的文件一般可以分为三份：

  1. 由 fastp 生成的干净的.fastq数据和.html质控报告。
  2. 由 SPAdes 生成中间计算过程文件，存放在spades_run/目录下。里的大多数文件是中间计算过程（如 K21, K33 等代表不同 K-mer 的尝试），平时基本不用管。其中根目录下的_ref_draft.fasta文件是从中复制出来的最终草图。
  3. quast_report/目录下存放的是 quast 对组装结果的质量报告，核心关注quast_report/report.html网页式汇总报告。
