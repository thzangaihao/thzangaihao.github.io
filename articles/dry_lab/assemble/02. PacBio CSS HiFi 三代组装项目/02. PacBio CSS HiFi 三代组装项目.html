02. PacBio CSS HiFi 三代组装项目

前言

这篇博客适用于由 PacBio CSS HiFi 三代测序平台产生的长读段数据在无参考证据文件时的从头组装（de novo组装）。因为组装流程已经被脚本高度自动化，这篇文章简单讲一下具体流程，环境搭建等，并解读软件的输出报告和意义。简单言之，我们是将公司给我们的.fastq文件组装出.fasta文件。

文件校准与预处理

数据完整性验证

md5 校验是文件完整性和准确性（防止被篡改）的常用方法，我们会根据文件算出一串哈希密钥，再根据公司提供的哈希密钥进行验证，若两者一致，则文件完整。

在 Linux 下，我们使用 md5sum 进行文件哈希密钥计算，而在 Windows 下，我们可以使用 WinMD5 进行验证。

数据质量评估

公司给我们的数据已经经过了 PacBio 官方软件 ccs 的处理，质量非常好，我们直接使用就好了，不需要再复洗数据了！。

如果你不放心，我们也可以使用 fasqp 等进行再次清洗评估（.bam 转 .fastq 见下文）。

组装

Hifiasm 组装

我们使用 Hifiasm 进行组装，Hifiasm 为 DBG 算法组装工具，这里只做脚本的备份：

hifi_assemble
文件格式：.py | 大小：7.4 KB

Hicanu 组装

Hicanu 是基于 OLC 算法的组装工具，组装效果也不错，但相较于 Hifiasm 来说，Hicanu 的运行时间更长一点。

这个脚本也支持 .bam 文件和 .fastq 文件的直接投喂。

历经数个月的开发与功能完善，这个脚本功能已经非常完善了！
