03. BLAST

2026-04-08 比较基因组学

1. 前言

BLAST (Basic Local Alignment Search Tool) 是一种基于启发式（Heuristic）算法的序列比对工具，旨在快速寻找序列间的局部相似性（Local Alignment）。关于启发式算法，我们会在算法栏进行讲解。本质上 BLAST 就是寻找相似序列。

在学习 BLAST 时，我们一定要明确 “用什么去比对什么” 根据查询序列（Query，就是我们要比较的序列）和数据库（Subject）的类型，BLAST 分为以下五种核心工具：

工具名称查询序列 (Query)数据库 (Subject)常用场景
blastn核苷酸 (DNA/RNA)核苷酸找同源基因、物种鉴定
blastp蛋白质蛋白质功能预测、结构域分析
blastx核苷酸 (6框翻译)蛋白质鉴定新测序 DNA 中的蛋白质编码基因
tblastn蛋白质核苷酸 (6框翻译)在基因组中寻找未标注的蛋白序列
tblastx核苷酸 (6框翻译)核苷酸 (6框翻译)亲缘关系较远的物种间寻找同源序列

2. BLAST 数据库

做 BLAST 分析需要用到数据库，在这里展示常用的数据库并作简单介绍：

2.1 大杂烩数据库

nt (Nucleotide Collection)

包含 GenBank、RefSeq、EMBL、DDBJ 等几乎所有公开的 DNA 序列。当你做核酸的 BLAST 时不知道选用哪个数据库时，先用这个数据库最好了！但是这个数据库数据量巨大，比对速度较慢，通常用于物种鉴定或寻找新基因。且容易出现大量 “假设蛋白” ，我去 BLAST 就是想知道这个基因的功能，结果 nt 数据库告诉你这个基因别人发现了，但也不知道干啥的，说了一句正确的废话。

nr (Non-redundant Protein)

去冗余后的蛋白质序列，就是在数据库中人类已知的所有蛋白序列，且去掉了功能和序列相似性的那部分冗余，就是蛋白质版本的 nt 库。现在 NCBI 推荐默认使用 ClusteredNR。它将相似蛋白聚类，速度比传统的 nr 快很多，且结果更具代表性。

2.2 精选与参考数据库

RefSeq (Reference Sequence)

每一个 ID 都是唯一的，代表了该物种的官方参考标准（如人、小鼠、拟南芥）。做物种间同源基因比对时的首选。

UniProtKB (Universal Protein Resource)

Swiss-Prot: 人工精选，注释信息极丰富且准确。

TrEMBL: 自动翻译，规模大但准确性稍逊。

2.3 比较基因组学专用平台

Ensembl / Ensembl Genomes

专注于脊椎动物、植物、菌类等。它不仅提供序列，还提供预先计算好的基因组对齐数据。

CoGe (Comparative Genomics)

这是一个专门的平台，允许你把自己的序列和成千上万个已发表的基因组进行大规模 BLAST (CoGeBlast)，并直接可视化基因组间的共线性。

PDB (Protein Data Bank)

如果你关注的是结构比较，BLAST 针对 PDB 的比对能告诉你这个蛋白在 3D 空间上长什么样。

4. BLAST 输出解读

4.1 重要参数

E-value

E-value (Expect value，期望值)，定义在随机背景下，匹配到当前得分序列的期望次数。

E 值越小越可靠。$E < 10^{-5}$：通常认为具有显著的同源性。$E$ 接近 0 说明匹配结果在统计学上几乎不可能由随机产生。

Identity

Identity (一致度/恒等度) 定义：Query 和 Subject 之间完全相同的碱基或氨基酸所占的比例（%）。

在比较基因组学中，蛋白质序列的 Identity 达到 30% 以上通常暗示功能相似。

Query Coverage

Query Coverage (查询序列覆盖度) 定义为 Query 序列中有多少比例参与了比对。

如果 Identity 很高但 Coverage 很低，说明可能只是由于某个保守的结构域（Domain）引起的局部匹配，而非全基因组水平的同源。

4.2 -outfmt 6

当你打开 outfmt 6 的结果文件时，你会看到 12 列数据，我们需要明白这几行：

  1. qseqid: Query 的名称
  2. sseqid: 匹配到的数据库序列（Subject）名称
  3. pident: Identity (%)，恒等度
  4. length: 比对长度
  5. mismatch: 错配数
  6. gapopen: 缺口数
  7. 中间是起止位置
  8. evalue: E 值（判定靠不靠谱的关键）
  9. bitscore: 比对得分（得分越高，匹配越好）
