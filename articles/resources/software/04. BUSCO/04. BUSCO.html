04. BUSCO

简介

BUSCO (Benchmarking Universal Single-Copy Orthologs) 通过比对单拷贝直系同源基因库，来量化组装的完整度。

BUSCO 的核心逻辑是：在一个给定的分类范围内（比如“真核生物”或“胚胎植物”），绝大多数物种都应该拥有且仅拥有一份特定的保守基因。如果你的组装结果里这些基因找得越全，说明组装质量越高。

环境搭建及软件安装


# 1. 创建并激活环境 (建议安装 v5.x 以上版本)
conda create -n busco -c conda-forge -c bioconda busco=5.8.1
conda activate busco

# 2. 检查是否安装成功
busco --version

用法

数据库选择及下载

你可以在 这里 看到 BUSCO 支持的所有的 orthoDB 数据库及库基因大小。如果你发现你研究的属（Genus）在 BUSCO 数据库列表中不存在，这是非常正常的现象。BUSCO 的 odb10 数据库并不是针对每一个具体的“属”设计的，而是基于进化上的类群进行层级化分布的。当找不到对应的属级数据库时，你应该遵循“向上寻根，择优匹配”的原则。如果你实在是找不到，你也可以用 eukaryota_odb10 库进行评估。

注意，当数据库选择的太广时，你的结果可能看起来非常漂亮（比如 99% 完整），但其实是因为它只检查了最基础的一小部分基因。如果选择错，结果会非常难看（大量 Missing），因为两个物种的基因组成完全不同。

脚本

这个脚本默认用 hypocreales_odb10 数据库进行分析，你可以在终端输入你样本的最优数据库进行评估（如果没有下载，则会先进行下载）。

BUSCO
文件格式：.py | 大小：5.0 KB
如果你已下好数据库，建议开启--offline；若未下载，请去掉此行：

cmd = [
    "busco",
    "-i", file_path,
    "-o", out_name,
    "-l", lineage,
    "-m", mode,
    "-c", cpu,
    "--offline"  # 如果你已下好数据库，建议开启；若未下载，请去掉此行
]

输出

  • C：98.8% (Complete): 代表在选择的数据库中的核心保守基因中，找齐了其中的 98.8%。

  • D: 0.1% (Duplicated): 有 0.1% 基因出现了重复。
  • F: 0.3% (Fragmented): 有 0.3% 的基因是碎裂的（只找到了一部分）。
  • M: 0.9% (Missing): 有 0.9% 基因完全没找到。
